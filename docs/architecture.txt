SagTask v2.2.1 — Architecture Reference
==========================================

This file is the canonical architecture diagram for SagTask. The README
includes a condensed version; this file is the full picture, including
context-injection layers, multi-profile layout, and data flow.

┌──────────────────────────────────────────────────────────────────────────────┐
│  CROSS-SESSION RECOVERY  (why SagTask survives gateway restarts)            │
│                                                                              │
│   Session 1                        Session 2                        Session N│
│   ─────────                        ─────────                        ────────│
│   Agent works on step 3.2          New chat / restart                New day│
│        │                                 │                                 │
│        ▼                                 ▼                                 ▼│
│   .sag_task_state.json          on_session_start hook          Agent picks up│
│   gets updated                  reads .active_tasks.json        exactly where│
│   + git commit                  restores _active_task_id        it left off  │
│        │                                 │                                 │
│        │            ALL ON DISK          │                                 │
│        └─────────────────────────────────┴─────────────────────────────────┘│
│                                                                              │
│   Key invariant: NOTHING important lives in RAM.  Every state change         │
│   is durably persisted to Git + .sag_task_state.json before the tool         │
│   returns success.  Gateway crash = at most one step lost, not the project. │

HOW TO READ THIS DIAGRAM
========================

- Read top-to-bottom for the request flow.
- Read top-section for "what lives where" (multi-profile, plugin, storage).
- Read middle-section for the internal plugin structure (20 tools, 3 groups).
- Read lower-middle for the advance lifecycle (plan → execute → verify → advance).
- Read bottom for the cross-session recovery guarantee.
